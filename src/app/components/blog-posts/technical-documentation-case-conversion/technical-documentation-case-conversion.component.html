API Endpoint and Parameter Naming Conventions

The Foundation of API Documentation

Consistent API naming conventions are crucial for developer adoption. Research shows that well-formatted API documentation can increase developer productivity by 40% and reduce support requests by 60%.

RESTful API Endpoint Conventions

Standard REST Endpoint Formatting:

Resource Endpoints (snake_case):
GET /api/v1/user_profiles
POST /api/v1/user_profiles
PUT /api/v1/user_profiles/{{ '{' }}id{{ '}' }}
DELETE /api/v1/user_profiles/{{ '{' }}id{{ '}' }}
  • Use snake_case for multi-word resource names
  • Keep endpoint names lowercase and descriptive
  • Use plural nouns for resource collections
  • Maintain consistent versioning in URLs

GraphQL Field Naming

GraphQL Convention (camelCase):

Query Fields:
userProfile
userProfiles
createUserProfile
updateUserProfile
  • Use camelCase for all GraphQL fields and operations
  • Start with lowercase for fields and mutations
  • Use descriptive, action-oriented names for mutations
  • Maintain consistency across schema definitions

Parameter and Variable Naming

Parameter Naming Standards:

Query Parameters:
?user_id=123&sort_by=created_at
?page_size=20&page_number=1
Path Parameters:
/users/{{ '{' }}user_id{{ '}' }}/profiles/{{ '{' }}profile_id{{ '}' }}

Code Example Formatting in Documentation

Function and Method Naming

Language-Specific Conventions:

JavaScript/TypeScript (camelCase):

Python (snake_case):

Variable and Constant Naming

Variable Naming Best Practices:

Constants (UPPER_SNAKE_CASE):
const API_BASE_URL = 'https://api.example.com'
const MAX_RETRY_ATTEMPTS = 3
Variables (camelCase/snake_case):
let userProfile = getUserProfile()
user_profile = get_user_profile()

Technical Writing Style Guides

Microsoft Style Guide

Microsoft Documentation Standards:

• UI Elements: Title Case for buttons, menus, and dialog boxes
• Code Elements: Monospace font for all code references
• File Names: Title Case for file and folder names
• Procedures: Sentence case for step descriptions

Google Developer Style Guide

Google Documentation Standards:

• API Names: camelCase for methods and properties
• Constants: UPPER_SNAKE_CASE for all constants
• Classes: PascalCase for class and interface names
• Variables: camelCase for local variables

Apple Human Interface Guidelines

Apple Documentation Standards:

• Menu Items: Title Case for all menu items
• Button Labels: Title Case for action buttons
• Dialog Text: Sentence case for dialog content
• Error Messages: Sentence case with clear language
